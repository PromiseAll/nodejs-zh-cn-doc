新增于: v1.6.0

在启动时预加载指定的模块。

遵循 require() 的模块解析规则。 module 可以是文件路径，也可以是 node 模块名称。

仅支持 CommonJS 模块。 尝试使用 --require 预加载 ES6 模块，则将失败并显示错误。

Added in: v1.6.0

Preload the specified module at startup.

Follows require()'s module resolution rules. module may be either a path to a file, or a node module name.

Only CommonJS modules are supported. Attempting to preload a ES6 Module using --require will fail with an error.
